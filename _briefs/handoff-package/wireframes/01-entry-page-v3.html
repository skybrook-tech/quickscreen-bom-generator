2 Live Google Maps · "Use this view" NEW

Map takes over. Native Google pan / zoom / satellite. No drawing yet.

Address geocoded → the customer drops into a fully interactive Google Maps view. Pan and zoom freely with the native Google controls. Toggle between satellite and roadmap. Tilt and rotate if useful. Find the angle that shows the fence line clearly — straight-on, slight tilt, full satellite, mix of satellite+roadmap. When the view's right, click "Use this view". That snapshot becomes the static canvas the calculator draws on.

NEW · live map first · capture-then-draw flow

Why live-firstv3

  • The current QuickScreen app already does this — the map IS Google Maps; user clicks "Use this view" to lock it
  • This is the only way to give a customer accurate roof/path/tree context — letting them choose the exact zoom + angle
  • Once captured, the snapshot is static (fast, no Google quota burn while drawing)
  • "Change view" link in the bar at top lets them go back and re-capture

Native Google controls visible

  • Bottom-right: native Google zoom (+/−)
  • Top-left: Satellite / Map type toggle (same style as Google Maps)
  • The "Live" pulse-dot pill top-right reminds the user this is interactive
  • No Anyfence drawing toolbar appears yet — clean Google interaction

"Use this view" CTA

  • Bottom-centre, ember orange — primary call to action
  • Sub-label: "Locks this view into your canvas"
  • Click → screenshot taken via Google's snapshot API → loaded into canvas as static layer → drawing tools appear in Stage 3

3 View captured · full toolbar lights up NEW

Snapshot locks. Drawing tools appear. Sidebar reveals fence types.

The moment "Use this view" fires, three things happen at once: the Google interactivity freezes (now a static satellite layer), the full DRAW / SITE / ACTIONS / VIEW toolbar appears across the top, and the sidebar slides in with the fence-type catalogue. The "Live" pill turns into a "🔒 Captured" pill so the customer knows the map is now drawing-ready, not panning.

📋 Tool inventory

Every drawing tool in the wireframe — what's shipping today vs proposed.

All ten existing tools from the live QuickScreen calc, plus six new ones I'd recommend adding to make the canvas genuinely useful as a site survey + quoting tool. Each new tool justified below.

DRAW · the fence + gate primitives

Already shipping in CanvasToolbar.tsx on master.

⟋
Draw Fence DSHIP
Tap corners to place fence-line nodes. Double-tap to finish a run.
⛩
Gate GSHIP
Drops a gate onto the nearest fence segment. Nudge to position.
✥
Move/Edit ESHIP
Drag nodes, reshape runs, reposition gates.
↶↷
Undo / Redo / Clear SHIP
Standard. Clear prompts confirmation.

Already shipping. Used to mark anything that's on the site but isn't a fence run.

SITE + · 3 approved new tools

Cut from 6 proposed down to 3 — these go into the SITE group alongside the existing ones.

📷
Photo pin ONEW
Drop a photo on a specific spot. Installer sees the photo before site visit (existing fence's condition, tricky corner, neighbour's structure). Makes walkthrough videos optional in many cases.
🌳
Tree RNEW
Circle marker for trees that affect fence runs (root impact, branches over the line). Helpful for installer prep.
⊕
North arrow NNEW
Auto-placed compass marker showing true north (Google Maps gives us the bearing). Standard for any site plan.
Dropped from v3 proposal: Measure (ruler), Council setback line, Vehicle turning circle.
If any of these turn out to matter later, easy to add back as a v4 increment.
